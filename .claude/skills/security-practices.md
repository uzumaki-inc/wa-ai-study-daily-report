# スキル: セキュリティプラクティス（公開リポジトリ）

このプロジェクトは**公開リポジトリ**です。全コードで以下を遵守してください。

## 絶対にコード/ドキュメントに書いてはいけないもの

- Slack Webhook URL
- Slack チャンネル名・チャンネルID
- Slack ワークスペース名・ワークスペースID
- Anthropic API Key
- 個人名・メールアドレス・社内URL
- その他、組織を特定できる情報

## 機密値の扱い

- すべて環境変数経由で注入する（`.env` / GitHub Secrets）
- `.env` は `.gitignore` に含める
- ドキュメント内の例示はプレースホルダーを使う（`#your-channel` 等）
- コード内で機密値を文字列結合してログ出力しない

## コーディング時のチェックリスト

1. ハードコードされた文字列にURLやキーが含まれていないか
2. ログ出力に機密値が含まれていないか
3. エラーメッセージに内部情報が露出していないか
4. 環境変数のバリデーションを起動時に行っているか
5. `npm audit` で脆弱性がないか

詳細は `docs/development-guidelines.md` のセキュリティ方針を参照。

# バグ修正: Webビュー生成のJSONパース失敗

> **ステータス**: 対応中
> **発生日**: 2026-03-31〜2026-04-03（4日連続）
> **影響**: Webビュー（「もっと見る」ページ）が生成されず404になる。Slack投稿は正常。

## 症状

- GitHub Actions実行は成功するが、`ARTICLES_JSON` のパースに失敗
- `[Claude] 全記事JSONのパースに失敗。Webビュー生成をスキップします` がログに出る
- JSON内容ログが途中で切れている（`"sou` で終了）
- 3/31〜4/3の4日連続で発生

## 原因分析

Claude APIのレスポンスで `</ARTICLES_JSON>` 閉じタグが出力されていない、
またはJSON自体が途中で切れている。

考えられる原因:
1. **max_tokensに到達** — 16384でも足りず、出力が途中で切れた
2. **プロンプトの出力量が多すぎ** — 62件の記事 × summaryで出力が膨大に
3. **stop_reason未チェック** — max_tokens到達を検知できていない

## 修正方針

- [ ] `stop_reason` をログ出力して原因を確定する
- [ ] `max_tokens` をさらに増やす or 記事数を減らす
- [ ] `</ARTICLES_JSON>` 閉じタグがない場合でもJSONの途中までパースを試みる
- [ ] ビルド・テスト・手動実行で動作確認
